# MoviePilot Plugins

个人维护的 MoviePilot V2 插件集合，仅用于自用测试。

在 MoviePilot 插件市场的第三方仓库设置中添加：

```text
https://github.com/g-steven037/MoviePilot-Plugins
```

当前包含：

- `P115RapidRetry`：监控硬链接目录，115秒传未命中时转移到临时目录并按 Cron 重试；秒传成功后安全删除对应本地硬链接。
- `EmbyLibraryCover`：读取 Emby 最新媒体海报，生成横版媒体库封面，并可按 Cron 选择性上传覆盖。

# MoviePilot 115 秒传重试（仅自用）

适用于 MoviePilot V2.14.2。插件只调用 115 上传初始化接口判断秒传，不包含完整上传代码。

## 安全处理流程

1. PT 下载目录是必填的受保护目录：仅用于验证隔离关系，绝不扫描、移动或删除。
2. 使用单工作线程和有界队列递归实时监控硬链接目录。
3. 只处理普通硬链接；拒绝单链接文件、符号链接、Junction 和其他重解析点。
4. 秒传成功前后核对设备号、文件 ID/inode、大小、修改时间和链接数，身份一致才删除。
5. 秒传失败后仅允许同文件系统 `os.replace()` 原子移动到临时目录，不进行跨盘复制。
6. 临时目录严格按 Cron 重试，每轮默认最多 10 个文件；普通未命中不叠加额外延迟。
7. 认证失败立即熔断；限流响应全局暂停一小时。
8. 历史只保存匿名任务 ID 和内部安全码，不保存路径、文件名、Cookie、响应体或原始异常。
9. 所有实时与重试请求共用全局风控闸门：默认至少间隔30秒，每小时最多30次秒传初始化操作。
10. 认证、限流、小时额度和连续失败熔断会持久化；重启插件或MoviePilot不会提前解除。

插件启用后，MoviePilot 日志默认记录文件名、来源目录、本地 SHA1 和秒传匹配结果，但始终不会记录 Cookie、115响应体或原始异常。

日志按处理阶段输出：开始计算 SHA1、计算完成或缓存命中、秒传结果、失败移入临时目录、Cron 重试以及成功删除。115接口原始响应可能包含用户 ID、pickcode、目标目录和回调参数，因此只记录脱敏后的内部状态码，不输出原始响应体。

## Cookie 安全边界

配置页使用密码框直接填写115 Cookie。插件只把 Cookie 交给115客户端用于访问115官方接口，不发送给其他第三方，不写入插件日志、历史记录或任务数据。MoviePilot 仍需将明文 Cookie 保存到自身配置数据库，浏览器提交配置时也会把它传给 MoviePilot 服务端；请启用 HTTPS、限制管理端访问并保护 MoviePilot 数据目录。

风控状态只保存由 MoviePilot `SECRET_KEY` 加键生成的不可逆 Cookie 标记、请求时间和固定状态码，不保存 Cookie 或可直接比对的普通哈希；若 MoviePilot 主密钥不可用，插件拒绝启动。

## 目录要求

- PT、硬链接、临时目录必须是三个互不包含的绝对路径。
- 三个目录必须位于同一文件系统；否则插件拒绝启动。
- 硬链接目录中的文件必须保持至少两个硬链接，避免误删唯一文件。
- 不要把 MoviePilot 配置目录、系统根目录或其他敏感目录配置为监控目录。
- 同一115账号只建议运行一个插件实例。小时额度与请求间隔在单个 MoviePilot 实例内持久化，多个容器或多台主机之间不会共享计数。

## 配置

- `115 Cookie`：直接填写，输入框以密码形式隐藏；必须包含 `UID` 和 `SEID`。
- `受保护的PT下载目录`：只校验，不扫描。
- `硬链接实时监控目录`：实时事件来源。
- `失败临时目录`：失败文件的原子移动目标。
- `115目标目录ID`：根目录为 `0`。
- `Cron`：临时目录重试计划。
- `立即运行秒传一次`：开启并保存后，立即扫描一次硬链接监控目录并提交当前文件；任务提交后开关自动复位。
- `立即重试秒传一次`：开启并保存后，立即扫描临时目录并执行一轮重试；任务提交后开关自动复位。
- `文件稳定等待秒数`：范围 1～3600。
- `每轮最大重试文件数`：范围 1～100。
- `单文件最大重试次数`：首次实时尝试不计入，范围 1～100；达到上限后停止自动重试。默认保留文件，是否删除由下方独立开关控制。
- `115请求最小间隔秒数`：范围5～300，默认30秒；实时首次请求和Cron重试共同遵守。
- `每小时最多115请求数`：滚动一小时额度，范围1～120，默认30；达到额度后暂停到最早一次请求退出统计窗口。
- `连续技术失败熔断次数`：范围2～20，默认5；只统计网络错误、超时、无效响应、客户端错误和API拒绝，正常未命中不计入。
- `连续失败暂停分钟数`：范围10～1440，默认60分钟。
- `重试耗尽后删除文件及空文件夹`：默认关闭。开启后，文件达到最大重试次数仍未秒传成功时，仅在身份校验通过后删除临时目录中的对应硬链接，并向上清理已经变空的父文件夹；临时根目录、PT源文件、非空文件夹以及身份发生变化的文件不会删除。更新前已经耗尽并保留的文件也会在后续 Cron 扫描中按“每轮最大重试文件数”逐批安全清理。
- `Bot通知`：使用 MoviePilot 已配置的消息渠道，并明确路由到 MoviePilot 超级管理员；秒传成功或达到最大重试次数时通知，不需要在插件中填写 Bot Token。成功通知显示实际重试次数，实时首次秒传成功为0次；临时目录重试成功为本次成功对应的累计重试次数。通知渠道必须启用“插件”类型，管理员账号也必须已绑定对应消息渠道的接收身份。插件日志会记录通知已提交或提交失败。
- `详细日志`：默认打开；关闭后使用简短日志，只记录秒传状态、文件名、当前/最大重试次数，以及转移后的临时文件夹。

秒传成功并安全删除对应文件后，插件会从文件所在目录开始向上清理空目录，并记录每个已删除目录。硬链接监控根目录和失败临时根目录永远不会被删除；非空目录、符号链接、Junction、其他重解析点或越界目录不会处理。

- `定时清理空文件夹`：默认关闭；开启后按独立 Cron 扫描指定根目录下的空文件夹。若触发时正在处理文件，本轮跳过，不延后补执行。
- `定时清理空文件夹根目录`：支持多个目录，每行填写一个已存在的容器内绝对路径；界面仅使用 `/path/to/cleanup-root-1`、`/path/to/cleanup-root-2` 作为通用格式示例，不代表真实目录。所有填写的根目录本身永远不会删除，也不得与受保护PT目录重叠。
- `空文件夹清理 Cron`：默认 `0 4 * * *`，即每天凌晨4点执行。

定时清理与实时秒传、失败转移、Cron重试共用同一操作锁，不会并发修改目录。文件处理期间到达清理时间时，本轮清理直接跳过，不延后补执行，等待下一次 Cron。已有秒传重试状态在插件更新或重启后仍会保留。秒传成功时始终先验证并删除文件，再清理其空父目录。每个定时扫描根目录最多检查100000个目录项，超过限制时仅跳过对应根目录。

两个“立即”功能不会绕过风控：仍使用同一个有界队列、单工作线程、操作锁、最小请求间隔、小时额度、熔断状态、每轮最大文件数和单文件最大重试次数。立即重试遇到正在执行的目录操作或已生效的风控熔断时会安全跳过，并在日志中记录固定状态码。

## 安全状态码

插件使用固定安全状态码替代115原始响应和异常文本，避免日志泄露账号或接口数据。常见状态码：

- `RAPID_SUCCESS`：115确认 SHA1 与文件大小匹配，秒传成功。
- `RAPID_MISS`：115未找到可复用内容，进入临时目录等待重试。
- `AUTH_FAILED`：Cookie 无效或登录失效，认证熔断。
- `RATE_LIMITED`：请求被限流，暂停请求。
- `NETWORK_ERROR`：网络访问失败。
- `FILE_CHANGED`：处理期间文件身份发生变化，停止删除或移动。
- `RETRY_EXHAUSTED`：达到配置的最大重试次数，停止自动重试。

# Emby 媒体库封面（仅自用）

适用于 MoviePilot V2.14.2。插件读取 Emby 媒体库及最新媒体海报，生成 `1920×1080` 横版封面。默认只生成图片，不会修改 Emby；只有主动开启“上传覆盖Emby封面”后才会上传。

## 功能

- `style_1`：背景图、标题和底部六张海报的经典横排样式。
- `style_2`：自动取色背景、标题和九张倾斜海报墙样式。
- `style_3`：参考横版媒体库卡片，使用横向背景、左侧电影感渐变和中英文标题；不在图片内部绘制圆角边框，避免与媒体库界面的卡片圆角叠加。短装饰线根据背景主色自动取色并增强亮度；没有背景图时自动拼接最多四张海报。
- 支持五段 Cron 和“立即运行一次”；定时任务只按照 Cron 表达式执行，同一时间只运行一个任务。
- 支持 JPG/PNG、内置中英文字体、自定义字体、输出目录和媒体库标题映射。
- 可选通过媒体库 ItemId、虚拟媒体库接口或两者上传封面。
- 插件页面保留最近100条处理结果，只记录媒体库名称和固定安全码。

## 配置

- `使用MoviePilot的Emby配置`：默认开启。通过MoviePilot官方 `MediaServerHelper` 读取已启用的Emby地址、API Key，并优先使用运行实例已解析的用户ID；敏感参数只在内存中使用，不复制到插件配置、日志或历史。
- `MoviePilot中的Emby服务器`：存在多个已启用Emby实例时可以指定；留空自动按名称选择第一个。
- `手动Emby地址/API Key/用户ID`：开启自动读取时隐藏；仅在关闭自动读取后显示并生效。地址必须是MoviePilot容器可访问的最终地址；API Key只通过 `X-Emby-Token` 请求头发送，不拼接到URL。
- 自动读取不会将MoviePilot保存的地址或API Key回填到手动输入框。配置页顶部会显示检测到的Emby服务器数量；选择服务器后，需要同时打开“插件启用”并保存，插件初始化时才会读取使用。
- `媒体库标题映射`：每行格式为 `媒体库名称|中文标题|英文标题`；英文标题需要换行时填写字面量 `\n`。
- `输出目录`：留空时写入 MoviePilot 配置目录下的 `plugins/embylibrarycover/output`；也可填写容器内可写的绝对路径。
- `中文/英文字体路径`：留空时中文使用内置的 MoviePilot CJK SC Bold（基于 Noto Sans CJK SC 的 GBK 字符子集），英文使用 Melete Bold；也可填写容器内已存在的 `.ttf`、`.otf` 或 `.ttc` 绝对路径覆盖默认字体。
- `上传覆盖Emby封面`：默认关闭。开启后才会向Emby写入图片。
- `上传后验证`：默认关闭；开启后会在上传前后各下载一次 ItemId 主图并比较本地SHA-256指纹，增加两次Emby请求。
- `校验HTTPS证书`：默认开启。除可信内网自签名环境外不应关闭。
- `画布与输出`：面板可设置输出宽高、JPG质量、格式、输出目录和中英文字体。画布限制为640×360至3840×2160且总像素不超过8294400。
- `标题排版对照设置`：Style 1 与 Style 2 重复的中文字号、英文字号、中英文垂直间距、英文字距和英文行距并排显示，便于对照调整；两种样式仍保存各自独立的值。
- `样式独立设置`：Style 1 的海报、渐变和雪花参数，以及 Style 2 的海报墙、旋转、位置、装饰条和背景颜色参数分别放置，不与其他样式混用。
- `Style 1参数`：标题坐标和海报尺寸采用内置默认值；面板保留字号、中英文垂直间距、英文字距/行距、海报数量/间距/Y坐标、遮罩、渐变和雪花数值。中英文间距默认 `20`，英文字距默认 `10`。
- `Style 2参数`：标题坐标和海报尺寸采用内置默认值；面板保留海报数量/间距/错位/旋转/中心坐标、字号、中英文垂直间距、英文字距/行距和颜色。中英文间距默认 `24`，英文字距默认 `6`。
- 上传目标固定为媒体库 ItemId；Style 1背景模糊/底部渐变/雪花及Style 2装饰条/自动取色均采用内置默认值，不再占用配置开关。
- 默认媒体库映射已按原始 `config.py` 收录动画电影、华语/外语电影、华语/外语剧集、日漫、国漫、综艺、儿童、纪录片和精选合集，名称必须与Emby实际媒体库完全一致。

## 安全边界

- 禁止HTTP跨域重定向，避免API Key被带到其他主机。
- 下载的单张图片最大30 MiB，JSON响应最大10 MiB，防止异常响应耗尽内存。
- 上传只执行明确配置的单一编码和接口目标，不轮询不同HTTP方法或载荷格式。
- Emby响应正文和原始异常不会写入日志；错误统一转换为固定安全码。
- 生成结果使用同目录临时文件和原子替换，避免留下半张封面；插件不会删除已有媒体文件。

## 字体许可

中文字体基于 Noto Fonts 官方 `NotoSansCJKsc-Bold.otf` 制作 GBK 字符子集并重命名为 `MoviePilotCJKsc-Bold.otf`，英文使用 `Melete-Bold.otf`，均按 SIL Open Font License 1.1 再分发。为兼容 MoviePilot v2.14.2 将二进制资源按UTF-8文本写入的安装器，两份字体以 gzip + Base64 文本保存，启动时在配置目录本地解码并校验 SHA-256；字体缺失、损坏或无法加载时插件会明确报错，不再静默回退生成方框。许可证位于 `fonts/OFL-NotoSansCJK.txt` 和 `fonts/OFL-Melete.txt`。
